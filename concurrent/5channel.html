1.概念 #

  • chanel 是由 Hoare 的 CSP 派生的同步原语之一。虽然它们可以用来同步内存访问，但它们最好用于 goroutine 之间传递信息。

  • 就像河流一样，一个 channel 充当着信息传送的管道，值可以沿着 channel 传递，然后在下游读出。出于这个原因，我们通常用 Stream 来做 chan 变量名的后缀。

2.channel 声明 #

go
// 声明一个channel.因为我们声明的类型是空接口，所以说它的类型是interface{}
var dataStream chan interface{}
// 使用内置的make函数实例化channel
dataStream = make(chan interface{})

3.声明单向 channel #

  • 声明一个单向 channel,只需包含 <- 操作符。

  • 声明和实例化一个只能读取的 channel,将 <- 放在左侧

go
var dataStream <- chan interface{}

  • 声明并创建一个只能发送的 channel, 将 <- 操作符放在右侧
go
var dataStream chan <- interface{}

  • 你通常不会看到单向 channel 实例化，但是会经常看到它们用作函数参数和返回类型，和我们看到的一样，这是非常有用的。因为当需要时，Go 语言会隐式地将双向 channel 转换为单向 channel

4.通道使用例子 #

go
package main

import "fmt"

func main() {
	stringStream := make(chan string)

	go func() {
		stringStream <- "Hello Channels!"
	}()

  // 为什么这里的代码，每次都在goroutine之后执行，难道是运气吗
  // 因为goroutine是阻塞的。这意味着只有channel内的数据被消费后，新的数据才能写入，而任何视图从空channel读取数的goroutine将等待至少一条数据被写入channel后才能读到。
	fmt.Println(<-stringStream)
}

5.channel 是阻塞的 #

  • goroutine 是阻塞的。这意味着只有 channel 内的数据被消费后，新的数据才能写入，而任何视图从空 channel 读取数的 goroutine 将等待至少一条数据被写入 channel 后才能读到。

6.通道中第二个返回的布尔值 #

  • value, ok := <- dataStream
  • 第二个返回值是读取操作的一种方式，用于表示该 channel 上有新数据写入，或者是由 closed channel 生成的默认值
  • 在程序中，能够提示 channel 中是否会有新的值写入是非常有用的。这有助于下游的程序知道什么时候消费，退出，给新的 channel 重新建立连接等。
  • 关闭 channel 是一个比较普通的操作，我们使用 close 关键字关闭一个 channel
go
package main

import "fmt"

func main() {
	intStream := make(chan int)
	close(intStream)

	integer, ok := <-intStream // ok是false,因为上面已经close(intStream)
	fmt.Printf("%v: %v", ok, integer)
}

  • 第二个返回值是 false,这表示我们收到的值是 int 或 0，而不是推到 stream 上的值

7.循环通道 #

  • 通过 range 关键字作为参数遍历(与 for 循环一起使用)，并且再 channel 关闭时自动中断循环。这允许对 channel 上的值进行简单的迭代，并且再 channel 关闭时自动中断循环，这允许对 channel 上的值进行简单的迭代
go
package main

import "fmt"

func main() {
	intStream := make(chan int)
	go func() {
		defer close(intStream) // 必须关闭，否则for循环哪里会死锁
		for i := 0; i < 5; i++ {
			intStream <- i
		}
	}()

  // 该循环不需要退出条件，并且range方法不返回第二个布尔值
	for integer := range intStream {
		fmt.Printf("%v", integer)
	}
}

8.通道达到广播效果 #

  • 关闭 channel 也是一种同时给多个 goroutine 发信号的方法。
  • 如果有 n 个 goroutine 在一个 channel 上等待，而不是在 channel 上写 n 次来打开每个 goroutine,你可以简单地关闭 channel
  • 由于一个被关闭的 channel 可以被无数次读取，所以不管有多少 goroutine 在等待它，关闭 channel 都比执行 n 次更适合，也更快。
go
package main

import (
	"fmt"
	"sync"
)

func main() {
	begin := make(chan interface{})
	var wg sync.WaitGroup
	for i := 0; i < 5; i++ {
		wg.Add(1) // wg计数+1
		go func(i int) {
			defer wg.Done() // wg计数-1
			<-begin         // 所有goroutine阻塞在这里
			fmt.Printf("%v has begun\n", i)
		}(i)
	}
	fmt.Println("Unblocking goroutines....")
	close(begin) // 关闭的时候，重新回到goroutine阻塞的地方执行
	wg.Wait()
}

9.提供有容量的 channel #

  • 这意味着即使没有在 channel 上执行读取操作，goroutine 依然可以执行 n 次写入。
  • 其中 n 是缓冲 channel 的容量
go
var dataStream chan interface{}
// 创建一个有4个容量的缓冲channel,这意味着我们可以把4个东西放到channel上，不管它是否被读取
dataStream = make(chan interface{}, 4)

10.无缓冲 channel #

  • 一个无缓冲的 channel 只是一个以 0 的容量创建的缓冲 channel
go
// 两个等价的channel, 这两个channel都是具有零容量的int channel
a := make(chan int)
b := make(chan int, 0)
1
2
3
  • 当我们讨论阻塞时，如果说 channel 是满的，那么写入 channel 阻塞。
  • 如果 channel 是空的，则从 channels 读取的是什么？
  • 无缓冲通道的 channel 的容量为 0，因此在任何写入之前 channel 已经满了
  • 一个没有下游接收的容量为 4 的缓冲 channel 在被写 4 次之后就满了，并且在写第 5 次的时候阻塞，因为没有其它地方放置第五个元素。
  • 缓冲 channel 是一个内存中的 FIFO 队列，用于并发进程进行通信

11.为什么无缓冲通道，可以放一个元素呢？ #

  • 如果一个缓冲 channel 是空的，并且有一个下游接收，那么缓冲区将被忽略，并且该值将直接送发送方传递到接收方。这在实践中是透明的，但是对了解缓冲 channel 的配置是值得的。

12.缓冲例子 #

go
package main

import (
	"bytes"
	"fmt"
	"os"
)

func main() {
	var stdoutBuffer bytes.Buffer
	defer stdoutBuffer.WriteTo(os.Stdout) // main函数结束之前，将stdoutBuffer的内容写到os.Stdout

	intStream := make(chan int, 4)

	go func() {
		defer close(intStream)
		defer fmt.Fprintln(&stdoutBuffer, "Producer Done.") // goroutine结束之前往stdoutBuffer写入`producer Done`
		for i := 0; i < 5; i++ {
			fmt.Fprintf(&stdoutBuffer, "Sending: %d\n", i) // 写入stdoutBuffer
			intStream <- i                                 // 我那个通道中放值
		}
	}()

	for integer := range intStream {
		fmt.Fprintf(&stdoutBuffer, "Received %v. \n", integer)
	}
}
